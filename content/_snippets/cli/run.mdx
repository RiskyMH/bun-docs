## CLI Usage

```bash title="terminal"
$ bun run <file or script>
```

### General Execution Options

<ParamField name="--silent" type="boolean">
	Don't print the script command
</ParamField>

<ParamField name="--if-present" type="boolean">
	Exit without an error if the entrypoint does not exist
</ParamField>

<ParamField name="--eval" type="string">
	Evaluate argument as a script. Alias: <code>-e</code>
</ParamField>

<ParamField name="--print" type="string">
	Evaluate argument as a script and print the result. Alias: <code>-p</code>
</ParamField>

<ParamField name="--help" type="boolean">
	Display this menu and exit. Alias: <code>-h</code>
</ParamField>

### Workspace Management

<ParamField name="--elide-lines" type="number" default="10">
	Number of lines of script output shown when using --filter (default: 10). Set to 0 to show all
	lines
</ParamField>

<ParamField name="--filter" type="string">
	Run a script in all workspace packages matching the pattern. Alias: <code>-F</code>
</ParamField>

<ParamField name="--workspaces" type="boolean">
	Run a script in all workspace packages (from the <code>workspaces</code> field in{' '}
	<code>package.json</code>)
</ParamField>

### Runtime &amp; Process Control

<ParamField name="--bun" type="boolean">
	Force a script or package to use Bun's runtime instead of Node.js (via symlinking node). Alias:{' '}
	<code>-b</code>
</ParamField>

<ParamField name="--shell" type="string">
	Control the shell used for <code>package.json</code> scripts. Supports either <code>bun</code> or{' '}
	<code>system</code>
</ParamField>

<ParamField name="--smol" type="boolean">
	Use less memory, but run garbage collection more often
</ParamField>

<ParamField name="--expose-gc" type="boolean">
	Expose <code>gc()</code> on the global object. Has no effect on <code>Bun.gc()</code>
</ParamField>

<ParamField name="--no-deprecation" type="boolean">
	Suppress all reporting of the custom deprecation
</ParamField>

<ParamField name="--throw-deprecation" type="boolean">
	Determine whether or not deprecation warnings result in errors
</ParamField>

<ParamField name="--title" type="string">
	Set the process title
</ParamField>

<ParamField name="--zero-fill-buffers" type="boolean">
	Boolean to force <code>Buffer.allocUnsafe(size)</code> to be zero-filled
</ParamField>

<ParamField name="--no-addons" type="boolean">
	Throw an error if <code>process.dlopen</code> is called, and disable export condition{' '}
	<code>node-addons</code>
</ParamField>

<ParamField name="--unhandled-rejections" type="string">
	One of <code>strict</code>, <code>throw</code>, <code>warn</code>, <code>none</code>, or{' '}
	<code>warn-with-error-code</code>
</ParamField>

<ParamField name="--console-depth" type="number" default="2">
	Set the default depth for <code>console.log</code> object inspection (default: 2)
</ParamField>

### Development Workflow

<ParamField name="--watch" type="boolean">
	Automatically restart the process on file change
</ParamField>

<ParamField name="--hot" type="boolean">
	Enable auto reload in the Bun runtime, test runner, or bundler
</ParamField>

<ParamField name="--no-clear-screen" type="boolean">
	Disable clearing the terminal screen on reload when --hot or --watch is enabled
</ParamField>

### Debugging

<ParamField name="--inspect" type="string">
	Activate Bun's debugger
</ParamField>

<ParamField name="--inspect-wait" type="string">
	Activate Bun's debugger, wait for a connection before executing
</ParamField>

<ParamField name="--inspect-brk" type="string">
	Activate Bun's debugger, set breakpoint on first line of code and wait
</ParamField>

### Dependency &amp; Module Resolution

<ParamField name="--preload" type="string">
	Import a module before other modules are loaded. Alias: <code>-r</code>
</ParamField>

<ParamField name="--require" type="string">
	Alias of --preload, for Node.js compatibility
</ParamField>

<ParamField name="--import" type="string">
	Alias of --preload, for Node.js compatibility
</ParamField>

<ParamField name="--no-install" type="boolean">
	Disable auto install in the Bun runtime
</ParamField>

<ParamField name="--install" type="string" default="auto">
	Configure auto-install behavior. One of <code>auto</code> (default, auto-installs when no
	node_modules), <code>fallback</code> (missing packages only), <code>force</code> (always)
</ParamField>

<ParamField name="-i" type="boolean">
	Auto-install dependencies during execution. Equivalent to --install=fallback
</ParamField>

<ParamField name="--prefer-offline" type="boolean">
	Skip staleness checks for packages in the Bun runtime and resolve from disk
</ParamField>

<ParamField name="--prefer-latest" type="boolean">
	Use the latest matching versions of packages in the Bun runtime, always checking npm
</ParamField>

<ParamField name="--conditions" type="string">
	Pass custom conditions to resolve
</ParamField>

<ParamField name="--main-fields" type="string">
	Main fields to lookup in <code>package.json</code>. Defaults to --target dependent
</ParamField>

<ParamField name="--preserve-symlinks" type="boolean">
	Preserve symlinks when resolving files
</ParamField>

<ParamField name="--preserve-symlinks-main" type="boolean">
	Preserve symlinks when resolving the main entry point
</ParamField>

<ParamField name="--extension-order" type="string" default=".tsx,.ts,.jsx,.js,.json">
	Defaults to: <code>.tsx,.ts,.jsx,.js,.json</code>
</ParamField>

### Transpilation &amp; Language Features

<ParamField name="--tsconfig-override" type="string">
	Specify custom <code>tsconfig.json</code>. Default <code>$cwd/tsconfig.json</code>
</ParamField>

<ParamField name="--define" type="string">
	Substitute K:V while parsing, e.g. <code>--define process.env.NODE_ENV:"development"</code>.
	Values are parsed as JSON. Alias: <code>-d</code>
</ParamField>

<ParamField name="--drop" type="string">
	Remove function calls, e.g. <code>--drop=console</code> removes all <code>console.*</code> calls
</ParamField>

<ParamField name="--loader" type="string">
	Parse files with <code>.ext:loader</code>, e.g. <code>--loader .js:jsx</code>. Valid loaders:{' '}
	<code>js</code>, <code>jsx</code>, <code>ts</code>, <code>tsx</code>, <code>json</code>,{' '}
	<code>toml</code>, <code>text</code>, <code>file</code>, <code>wasm</code>, <code>napi</code>.
	Alias: <code>-l</code>
</ParamField>

<ParamField name="--no-macros" type="boolean">
	Disable macros from being executed in the bundler, transpiler and runtime
</ParamField>

<ParamField name="--jsx-factory" type="string">
	Changes the function called when compiling JSX elements using the classic JSX runtime
</ParamField>

<ParamField name="--jsx-fragment" type="string">
	Changes the function called when compiling JSX fragments
</ParamField>

<ParamField name="--jsx-import-source" type="string" default="react">
	Declares the module specifier to be used for importing the jsx and jsxs factory functions.
	Default: <code>react</code>
</ParamField>

<ParamField name="--jsx-runtime" type="string" default="automatic">
	<code>automatic</code> (default) or <code>classic</code>
</ParamField>

<ParamField name="--jsx-side-effects" type="boolean">
	Treat JSX elements as having side effects (disable pure annotations)
</ParamField>

<ParamField name="--ignore-dce-annotations" type="boolean">
	Ignore tree-shaking annotations such as <code>@__PURE__</code>
</ParamField>

### Networking &amp; Security

<ParamField name="--port" type="number">
	Set the default port for <code>Bun.serve</code>
</ParamField>

<ParamField name="--fetch-preconnect" type="string">
	Preconnect to a URL while code is loading
</ParamField>

<ParamField name="--max-http-header-size" type="number" default="16384">
	Set the maximum size of HTTP headers in bytes. Default is 16KiB
</ParamField>

<ParamField name="--dns-result-order" type="string" default="verbatim">
	Set the default order of DNS lookup results. Valid orders: <code>verbatim</code> (default),{' '}
	<code>ipv4first</code>, <code>ipv6first</code>
</ParamField>

<ParamField name="--use-system-ca" type="boolean">
	Use the system's trusted certificate authorities
</ParamField>

<ParamField name="--use-openssl-ca" type="boolean">
	Use OpenSSL's default CA store
</ParamField>

<ParamField name="--use-bundled-ca" type="boolean">
	Use bundled CA store
</ParamField>

<ParamField name="--redis-preconnect" type="boolean">
	Preconnect to <code>$REDIS_URL</code> at startup
</ParamField>

<ParamField name="--sql-preconnect" type="boolean">
	Preconnect to PostgreSQL at startup
</ParamField>

<ParamField name="--user-agent" type="string">
	Set the default User-Agent header for HTTP requests
</ParamField>

### Global Configuration &amp; Context

<ParamField name="--env-file" type="string">
	Load environment variables from the specified file(s)
</ParamField>

<ParamField name="--cwd" type="string">
	Absolute path to resolve files &amp; entry points from. This just changes the process' cwd
</ParamField>

<ParamField name="--config" type="string">
	Specify path to Bun config file. Default <code>$cwd/bunfig.toml</code>. Alias: <code>-c</code>
</ParamField>

## Examples

Run a JavaScript or TypeScript file:

```bash
$ bun run ./index.js
$ bun run ./index.tsx
```

Run a package.json script:

```bash
$ bun run dev
$ bun run lint
```
